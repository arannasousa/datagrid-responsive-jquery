Inicialização.

A inicialização começa com a chamada aos componentes na tag HEAD do código html.

<!DOCTYPE html>
<html>
<head>
    <title>Pagina exemplo</title>

    <!-- IMPORTACAO BIBLIOTECA -->
    <link href="datagrid.css" rel="stylesheet"/>
    <script src="datagrid.js"></script>
</head>
<body>
<div class="espaco-datagrid"></div>
</body>
</html>

Para carregar o datagrid na div chamada espaco-datagrid, é usado o seguinte código dentro da HEAD.

<script>
    $(document).ready(function () {
    var x = $("div.espaco-datagrid").datagrid_tins({
        autoAjuste: true,        // largura ou tamanho de forma automatica
        url: "datagrid.json",    // url
        csrfmiddlewaretoken: '',
        colunas: [
            ["campo1", "Código"],   // ["nome_do_seu_campo", "Título"]
            ["campo2", "Primeiro nome"],
            ["campo3", "Último nome"],
            ["campo4", "Estado"],
            ["campo5", "e-Mail"],
            ["campo6", "Ativo?"],
            ["campo7", "Ações"]
        ]
        });
    });
</script>

Demonstração:

O modelo padrão vem configurado para atender até 4 tamanhos de acordo à largura do local onde o datagrid está. Esse tamanho define o ponto máximo para executar a determinada largura

        ...
        limiteTamanhoLarga: 2000,       // valores em 'px' (pixel)
        limiteTamanhoGrande: 1024,
        limiteTamanhoMedia: 970,
        limiteTamanhoPequena: 750
        ...

Configurando a aparição dos botões por tamanho do DATAGRID:

É possível configurar os botões que devem aparecer para cada tamanho do datagrid

Os botões são organizados em grupos:

  • grupoNavegacao: composto pelos botões de navegação. Esse grupo, por padrão, usa a classe "grupoCollapsePequena" e faz com que, só sejam escondidos quando a tela for menor que o tamanho definido em limiteTamanhoPequena.

  • grupoPesquisa: composto pela caixa de entrada de pesquisa e pelo botão de pesquisa. Esse grupo, por padrão, usa a classe "grupoCollapseMedia" e faz com que, só sejam escondidos quando a tela for menor que o tamanho definido em limiteTamanhoMedia.

  • grupoAtualizar: composto pelo botão atualização. Esse grupo, por padrão, usa a classe "grupoCollapseGrande" e faz com que, só sejam escondidos quando a tela for menor que o tamanho definido em limiteTamanhoGrande.

  • grupoOrdenacao: composto pelo de ordenação do datagrid. Esse grupo, por padrão, usa a classe "grupoCollapseGrande" e faz com que, só sejam escondidos quando a tela for menor que o tamanho definido em limiteTamanhoGrande.

  • grupoTela: composto pelo de configuração de exibição do datagrid (configura quantas colunas de informações devem aparecer no datagrid). Esse grupo, por padrão, usa a classe "grupoCollapseLarga" e faz com que, só sejam escondidos quando a tela for menor que o tamanho definido em limiteTamanhoLarga.

É possível trocar o modo de quando um grupo deve aparecer ou ficar oculto, trocando as classes padrão de cada um. Exemplo, os grupos originais são: 

        ...
        grupoNavegacao: "grupoCollapsePequena",
        grupoPesquisa:  "grupoCollapseMedia",
        grupoAtualizar: "grupoCollapseGrande",
        grupoOrdenacao: "grupoCollapseGrande",
        grupoTela:      "grupoCollapseLarga",
        ...

Trocando o grupo grupoPesquisa para esconder somente quando o tamanho do datagrid for menor que limiteTamanhoPequena, ao invés de menor do que limiteTamanhoMedia 

        ...
        grupoNavegacao: "grupoCollapsePequena",
        grupoPesquisa:  "grupoCollapsePequena",     // troca
        grupoAtualizar: "grupoCollapseGrande",
        grupoOrdenacao: "grupoCollapseGrande",
        grupoTela:      "grupoCollapseLarga",
        ...

Adição de botões do usuário aos grupos:

É possível inserir um botão em qualquer um dos grupos existentes. Existem 4 variáveis e cada é do tipo list ('[]'). Abaxo segue um exemplo.

1º - seu botão deve possuir o método .renderizar() e retornar um elemento "DOM".

var botaoExtra = {
    botao: {
        nome:   'botaoQualquer',
        titulo: 'Botão Extra',
        acao: function(){
            alert('teste de adição de botão')
        },
        icone:  '<i class="glyphicon glyphicon-music"></i>'
    },
    renderizar : function(){
        var b = $("<button>");
        b.attr({
            "name":     this.botao.nome,
            "type":     "button",
            "class":    "btn btn-danger"
        });
        b.append(
                this.botao.icone,
                this.botao.titulo
        ).click(this.botao.acao);

        return $("<div>").addClass("btn-group").append(b);
    }
}

2º - ao inicializar o datagrid, deve informar em qual grupo o seu botão vai ficar. No exemplo abaixo nosso botão vai ficar no grupo grupoCollapsePequena.

var datagridExemplo = $("div.espaco-datagrid-1").datagrid_tins({
    autoAjuste: false,
    tamanhoPadrao: "media",
    url: "url",
    identificacao:  "espaco-datagrid-1", // identificacao do datagrid
    limiteTamanhoLarga: 2000,
    limiteTamanhoGrande: 1024,
    limiteTamanhoMedia: 700,
    limiteTamanhoPequena: 600,
    colunas: [
        ["campo1", "Código"],
        ["campo2", "Primeiro nome"],
        ["campo3", "Último nome"],
        ["campo4", "Estado"],
        ["campo5", "e-Mail"],
        ["campo6", "Ativo?"],
        ["campo7", "Ações"]
    ],
    grupoCollapsePequena: [
            botaoExtra          // podem haver mais, já que é uma lista
    ]
});

Opções de inicialização:

Ao chamar a função datagrid_tins é possível fazer algumas configurações. Abaixo é exibida todas as opções e em seguida, é explanado cada item.

.datagrid_tins({
    identificacao: null,
    autoAjuste: true,
    metodo: 'POST',

    classesTamanhoTabela: {
        ' ':        "Automático",
        'larga':    "4 colunas",
        'grande':   "3 colunas",
        'media':    "2 colunas",
        'pequena':  "1 coluna"
    },
    tamanhoPadrao: ' ',

    limiteTamanhoLarga: 2000,
    limiteTamanhoGrande: 1024,
    limiteTamanhoMedia: 970,
    limiteTamanhoPequena: 750,

    grupoNavegacao: "grupoCollapsePequena",
    grupoPesquisa: "grupoCollapseMedia",
    grupoAtualizar: "grupoCollapseGrande",
    grupoOrdenacao: "grupoCollapseGrande",
    grupoTela: "grupoCollapseLarga",

    grupoCollapsePequena : [],
    grupoCollapseMedia : [],
    grupoCollapseGrande : [],
    grupoCollapseLarga : [],

    direcao: '-',
    direcaoLabel: {
        '': 'Cresc -> Decres',
        '-': 'Decres -> Cresc'
    },

    totalPaginas: 1,
    quantidadePorPagina: 30,
    pagina: 1,
    ordenadoPor: null,
    colunas: null,
    url: null,
    buscar: "",
    dados: {},

    carregamentoRequisicao: "Carregando dados!",
    semDadosRequisicao: "Sem dados para serem exibidos!",
    erroRequisicao: "Um erro aconteceu ao tentar solicitar sua requisição!",
    funcaoRenderizadora: null,

    antesCarregados: null,
    depoisCarregados: null,

    csrfmiddlewaretoken: ''
});

identificacao
Valor padrão: null.
Recebe uma 'string' e funciona como uma ID, identificando de forma única o datagrid.
Se não informado, ele será gerado automaticamente.

autoAjuste
Valor padrão: true.
Recebe um valor 'boolean' (true ou false).
Ele indica se o datagrid fará a detecção automática da sua própria largura ou não. Assim os botões serão escondidos, ou não, de acordo o tamanho do lugar onde o datagrid encontra-se.

Quando o valor for false, o datadrid é configurado num tamanho fixo indicado pelo campo tamanhoPadrao.

metodo
Valor padrão: 'POST'.
Recebe uma 'string' ('GET' ou 'POST').
Ele indica se o datagrid fará uma requisição via GET ou POST.

classesTamanhoTabela: {
  ' ': "Automático",
  'larga': "4 colunas",
  'grande': "3 colunas",
  'media': "2 colunas",
  'pequena': "1 coluna"
}
Valor padrão: dicionário.
Recebe um 'dicionário' com chaves padrão já definidas.
Nesse campo, a edição é feita nos campos à frente das chaves (' ', 'larga', 'grande', 'media', 'pequena').

São nesses espaços que devem ser informados os textos que irão aparecer no botão de configuração de exibição do datagrid - último botão

Quando o valor for false, o datadrid é configurado num tamanho fixo indicado pelo campo tamanhoPadrao.
